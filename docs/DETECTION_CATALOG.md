# Detection Catalog

The generated catalog contains 2,430 safe verification checks across frontend,
backend API, database audit, and CI/CD categories.

## Category Examples

- Frontend static bundle review
- Session and cookie token posture
- API schema and authorization review
- Database read-only audit posture
- CI/CD secret release and supply-chain checks

## Check Shape

Each check contains:

- ID
- Title
- Category
- Target type
- Execution mode
- Risk if found
- Confidence
- Destructive flag
- Evidence types
- Preconditions
- Remediation template
- Mapped standards

## Safety

Generated checks are metadata and orchestration guidance. They do not contain
exploit payload catalogs or destructive procedures.
